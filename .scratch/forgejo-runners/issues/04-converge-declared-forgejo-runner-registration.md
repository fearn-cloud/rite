# Converge declared Forgejo runner registration

Status: ready-for-agent
Category: enhancement

## What to build

Make Forgejo runner registration reproducible from declared state so the stateless Runner VM can be rebuilt and configured without preserving an irreplaceable manual registration file. Re-running configuration should register or refresh the instance-level runner without accumulating duplicate runners; a manual-token bootstrap may exist only as an explicit temporary fallback if the deployed Forgejo CLI path cannot fully support offline registration.

## Acceptance criteria

- [ ] Runner registration identity is declared or derived from declared state rather than treated as hand-created VM-local state.
- [ ] Forgejo-side configuration can create or refresh the instance-level runner registration used by the Runner VM.
- [ ] Re-running configuration is idempotent and does not create duplicate active runners for the same declared Runner VM.
- [ ] Runner labels in Forgejo match the declared VM-level runtime intent.
- [ ] The Runner VM can be rebuilt and reconfigured without restoring a backup to recover runner registration.
- [ ] Any manual-token fallback is documented as temporary and does not become the preferred steady-state path.
- [ ] Automated tests cover idempotent registration planning or rendering, duplicate avoidance, and runner label convergence.
