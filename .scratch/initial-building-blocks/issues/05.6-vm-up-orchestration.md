## What to build

Replace the `vm-up` stub with the unified operator workflow. `just vm-up vm=<name>` wraps safe scripts that run Prepare, selected-VM tofu plan with explicit approval, selected-VM tofu apply, and Configure.

## Acceptance criteria

- [x] `just vm-up vm=<name>` invokes a real script such as `scripts/vm-up <name>`.
- [x] The script rejects undeclared VMs before any mutation.
- [x] The script runs VM Prepare first.
- [x] The script runs selected-VM tofu plan through the tofu wrapper.
- [x] The script requires explicit operator approval after showing the plan and before apply.
- [x] The script runs selected-VM tofu apply through the tofu wrapper only after approval.
- [x] The script runs VM Configure after successful apply.
- [x] Approval safety lives in the script, not only in the just recipe.
- [x] `just` remains the unified operator interface while scripts remain safe direct debugging entry points.
- [x] Tests use fake Prepare, fake tofu wrapper, and fake Configure commands to verify order, refusal, approval denial, and approval success.
- [x] A failed Prepare, plan, apply, or Configure stops the sequence and reports the failed phase.

## Design decisions

- The operator-facing command is `just vm-up vm=<name>`.
- The script owns the safety semantics so direct script invocation is not a footgun.
- The selected-VM apply path avoids accidentally applying every VM in inventory during early rollout.
