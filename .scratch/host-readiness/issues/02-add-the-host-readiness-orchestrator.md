## What to build

Add Host Readiness as the first-class Host-level operator workflow. The workflow should be a resumable wrapper over existing commands, not a new provisioning engine.

Expose:

```sh
just host-up <host> endpoint=all auto_confirm=false keep_on_fail=false
scripts/host-up <host> [endpoint=<nas-endpoint|all>] [auto_confirm=<true|false>] [keep_on_fail=<true|false>]
```

For a selected Host, Host Readiness should perform or validate Bootstrap, run full Host Configure, build every Template declared for the Host, verify every declared Template on the Host, then run NFS shared-mount and Service-layer Acceptance Tests across every selected NAS Endpoint and every declared Template.

## Acceptance criteria

- [ ] `just host-up <host>` invokes `scripts/host-up <host>` with default `endpoint=all`, `auto_confirm=false`, and `keep_on_fail=false`.
- [ ] `scripts/host-up` rejects `host=<name>` as the positional Host argument and prints the matching `just host-up <name>` guidance.
- [ ] The script validates that the Host exists in Inventory before running any downstream workflow.
- [ ] If the Host Sibling SOPS File is missing, the script runs `scripts/host-bootstrap <host>`.
- [ ] If the Host Sibling SOPS File exists, the script verifies that it decrypts and contains `ssh_keys.bootstrap.private_key`.
- [ ] If Bootstrap is already satisfied, the script proves Host SSH reachability by running `scripts/host-shell <host> -- true` before continuing.
- [ ] Ambiguous Bootstrap state, such as an undecryptable Host Sibling SOPS File or one missing `ssh_keys.bootstrap.private_key`, stops the workflow before Configure.
- [ ] The script runs full Host Configure with all known Host Configure tags in the required order.
- [ ] The script fails Host Readiness when the Host declares no Templates.
- [ ] The script runs `scripts/templates-build <host>` before any Template Verification.
- [ ] The script verifies every Template declared under the selected Host's `proxmox.templates`.
- [ ] `endpoint=all` resolves every declared NAS Endpoint from Inventory.
- [ ] `endpoint=<name>` scopes acceptance to exactly the selected declared NAS Endpoint.
- [ ] Unknown endpoint names fail before any Acceptance Test starts.
- [ ] For every selected NAS Endpoint and every declared Template, the script runs NFS shared-mount acceptance and Service-layer acceptance.
- [ ] `auto_confirm=true` passes through only to downstream workflows that support non-interactive confirmation.
- [ ] `keep_on_fail=true` passes through to Template Verification and both Acceptance Test workflows.
- [ ] Bootstrap, Configure, Template Build, and Template Verification failures stop immediately.
- [ ] Acceptance failures continue across independent Template x NAS Endpoint cells unless `keep_on_fail=true` preserves artifacts that would collide with later cells.
- [ ] The script exits nonzero if any phase or acceptance cell fails.
- [ ] The script always prints a compact phase summary, including skipped/satisfied Bootstrap, full Configure, Template Build, each Template Verification, each acceptance cell, and final passed/failed status.
- [ ] Fast workflow tests cover command signature, just recipe wiring, Bootstrap missing/satisfied/ambiguous paths, Host reachability proof, full Configure tag invocation, no-Template failure, Template Build and Verification ordering, endpoint resolution, acceptance matrix expansion, `auto_confirm`, `keep_on_fail`, first-prerequisite-failure stopping, acceptance failure aggregation, and summary output.
