# Host Readiness composes host and acceptance workflows

Host Readiness is a first-class operator workflow, but it is a resumable wrapper over existing workflow commands rather than a separate provisioning engine. It treats an already-completed Bootstrap as a satisfied prerequisite only after proving SSH reachability with the stored per-Host credential, while preserving Bootstrap itself as one-shot by refusal; it then runs the full Configure scope, builds and verifies every Template declared for the Host, and runs NFS shared-mount and Service-layer Acceptance Tests across every selected NAS Endpoint and declared Template. This keeps `host-up` aligned with `vm-up` and `service-launch`: the wrapper proves readiness by composing the lower-level ceremonies instead of hiding new mutation paths behind a second implementation.
