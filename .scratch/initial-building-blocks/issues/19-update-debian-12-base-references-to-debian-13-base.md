**Category:** enhancement
**Summary:** Retire `debian-12-base` references in favor of `debian-13-base`

**Current behavior:**
The repository still contains a mix of Debian 12 and Debian 13 Template names. The live Host and VM inventory uses `debian-13-base`, but documentation, workflow tests, schema fixtures, inventory fixtures, and the old Debian 12 Template declaration still refer to `debian-12-base`. This makes the examples and tests drift from the current base Template.

**Desired behavior:**
`debian-13-base` should be the canonical Debian base Template name across the repository. Operator-facing documentation, architecture examples, workflow examples, tests, fixtures, and validation examples should use `debian-13-base` unless a test is intentionally exercising mismatched or missing Template behavior. The obsolete Debian 12 Template declaration should no longer remain as a first-class inventory Template if nothing is meant to build or verify it.

**Key interfaces:**
- Template inventory declarations — the canonical Debian base Template should be named `debian-13-base` and carry the current Debian 13 Cloud Image recipe.
- Host inventory `proxmox.templates` lists — examples and fixtures should list `debian-13-base` where they represent a normal valid inventory.
- VM inventory `placement.template` values — examples and fixtures should point at `debian-13-base` where they represent a normal valid VM.
- Operator workflows that accept a `template=<template>` argument — tests and docs should demonstrate `debian-13-base` as the current default example.
- Cross-file inventory validation tests — assertions and fixture setup should continue to prove Host-to-Template and VM-to-Template validation, using Debian 13 names for normal valid cases.

**Acceptance criteria:**
- [ ] A repository-wide search for `debian-12-base` has no remaining matches except cases that are explicitly documented in-test as intentional legacy or negative-case coverage.
- [ ] Operator-facing docs and architecture examples use `debian-13-base` in Template, Host, VM, and workflow examples.
- [ ] Valid inventory fixtures use `debian-13-base` consistently across Host, VM, and Template declarations.
- [ ] Workflow tests that create temporary Template, Host, or VM inventory use `debian-13-base` for the normal happy path.
- [ ] Negative validation tests still cover missing or mismatched Template behavior after the rename.
- [ ] The full automated test suite, or the relevant inventory/template/workflow test subset, passes after the change.

**Out of scope:**
- Changing the Debian 13 Cloud Image URL or checksum unless the current declared recipe is invalid.
- Adding a Debian 12 compatibility alias.
- Running live Proxmox template builds or verification on real Hosts.
