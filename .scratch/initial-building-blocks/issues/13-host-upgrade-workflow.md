## Agent Brief

**Category:** enhancement
**Summary:** Add the operator-visible Host upgrade workflow and runbook, separate from Host Configure.

**Current behavior:**
Host Configure is complete and is intentionally not the place for ordinary Proxmox package upgrade choreography. The issue is no longer blocked by the Host Configure slice, but the repo still lacks a dedicated Host upgrade operator surface and runbook.

**Desired behavior:**
The Operator has an explicit Host upgrade workflow, separate from `just host-configure`, that documents and executes the safe Proxmox package upgrade ceremony for one Host at a time. It should perform preflight checks, show downtime and backup expectations, run the package upgrade sequence only when explicitly invoked, report whether a reboot is required, and never reboot automatically.

**Key interfaces:**

- `just host-upgrade host=<host>` or an equivalent explicit operator command.
- A backing `scripts/host-upgrade <host>` workflow that uses existing Host identity, SSH, and inventory conventions.
- `runbooks/host-upgrade.md` as the durable human procedure.
- Host Configure documentation, which should point to the Host upgrade workflow rather than implying `host-configure` performs package upgrades.

**Acceptance criteria:**

- [ ] `runbooks/host-upgrade.md` documents preflight checks, VM downtime expectations, backup/snapshot expectations, upgrade command sequence, reboot decision points, and rollback/recovery notes.
- [ ] Upgrade workflow is exposed separately from `just host-configure`.
- [ ] Upgrade workflow requires explicit operator invocation per Host.
- [ ] Upgrade workflow reports whether an operator-controlled reboot is required.
- [ ] Upgrade workflow never reboots automatically.
- [ ] Host Configure documentation links to the upgrade workflow as the place for hypervisor package upgrades.
- [ ] Tests cover command wiring and the no-automatic-reboot contract.

**Out of scope:**

- Running a live Proxmox upgrade during implementation.
- Changing Host Configure package behavior except for documentation links.
- Automating rollback after a failed hypervisor upgrade.
