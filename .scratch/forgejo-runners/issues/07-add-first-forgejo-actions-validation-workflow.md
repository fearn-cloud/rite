# Add the first Forgejo Actions validation workflow

Status: ready-for-agent
Category: enhancement

## What to build

Add the first Forgejo Actions workflow that proves the Runner VM end to end using explicit validation runner labels. The workflow should run non-mutating repository health checks only and must not require SOPS decryption, live infrastructure credentials, direct Host or VM access, or deployment authority.

## Acceptance criteria

- [ ] A Forgejo Actions workflow runs on the declared validation runner labels.
- [ ] The workflow performs meaningful repository validation such as tests, schema checks, docs checks, or static checks.
- [ ] The workflow does not decrypt SOPS secrets.
- [ ] The workflow does not call live infrastructure apply or convergence commands.
- [ ] The workflow does not require SSH or API credentials for Hosts, VMs, NAS, PBS, or Services.
- [ ] The workflow proves that one job can execute in the Podman-backed Forgejo Runner Runtime.
- [ ] Documentation tells the Operator how to trigger and inspect the first validation workflow.
- [ ] Tests or static checks protect the workflow from drifting into phase-one-forbidden live infrastructure behavior where practical.
