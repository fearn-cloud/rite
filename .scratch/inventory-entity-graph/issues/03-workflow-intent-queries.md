# Inventory Entity Graph workflow intent queries

## What to build

Extend the Inventory Entity Graph with workflow intent queries for operator ceremonies that currently assemble cross-Entity facts by hand.

The graph should provide read-only intent facts for Service Launch, VM Lifecycle selected-host/provider planning, Template Verification, and Acceptance Policy resolution while preserving the ADR decision that workflow wrappers compose existing ceremonies rather than becoming new deployment engines.

## Acceptance criteria

- [ ] Extend the Inventory Entity Graph rather than creating a workflow-specific replacement model.
- [ ] Add graph queries for Service Launch intent, including the declared Service, Backend VM name, and whether Ingress Regeneration is required.
- [ ] Add graph queries for VM Lifecycle selected-host/provider facts, including VM placement Host and provider Host coverage facts needed by selected-VM OpenTofu workflows.
- [ ] Add graph queries for Template Verification intent, including selected Host, Template, management address, storage, static address, and matching Host bridge facts.
- [ ] Add graph queries for Acceptance Policy intent, including selected Host, Template, NAS Endpoint, generated Operational VM declarations, Ephemeral Dataset facts, and matching Host bridge facts.
- [ ] Public graph methods continue to accept Entity names rather than raw Entity dictionaries.
- [ ] Ordinary absence returns `None` where absence is valid; invalid or ambiguous declared workflow facts raise the domain-specific Inventory Entity Graph error.
- [ ] Migrate at least one workflow caller to prove the Interface reduces raw YAML traversal without changing the workflow's external behavior.
- [ ] Tests cover the new workflow intent queries directly and preserve workflow-level regression coverage for migrated callers.
- [ ] ADR-0027 and ADR-0028 remain intact: Service Launch and Host Readiness continue composing existing workflows instead of hiding new mutation paths.
