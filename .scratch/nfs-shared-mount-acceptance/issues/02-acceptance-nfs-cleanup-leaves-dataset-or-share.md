## Draft Agent Brief

**Category:** bug
**Summary:** Make NFS shared-mount Acceptance Test cleanup prove and report removal of the acceptance Ephemeral Dataset and Derived NFS Share.

**Current behavior:**
The NFS shared-mount Acceptance Test is intended to clean up generated Operational VMs, then remove the acceptance-owned NAS resources. A live run appears to leave the acceptance Dataset or Share behind. Existing automated tests only prove that the cleanup command is invoked; they do not prove live cleanup postconditions or guard against a cleanup plan that is too broad, too narrow, or silently ineffective.

**Desired behavior:**
After a successful default run, the Acceptance Test should leave no generated VMs, generated VM inventory artifacts, acceptance Ephemeral Dataset, or acceptance Derived NFS Share behind. If cleanup cannot prove that the Dataset and Share are acceptance-owned and safe to remove, the workflow should fail loudly and report exactly what was preserved and why. Cleanup must remain conservative: it must not delete adopted Datasets, unmanaged Shares, or unrelated fortress-owned Shares.

**Key interfaces:**

- NFS shared-mount Acceptance Test cleanup behavior — should destroy generated VMs and then remove only NAS resources owned by this acceptance workflow.
- NAS Reconcile acceptance destroy mode — should produce delete actions for marked Ephemeral Datasets and matching fortress-owned Shares, and should report blockers for unmarked or unmanaged overlaps.
- TrueNAS reality and apply behavior — should preserve and read fortress ownership markers consistently enough for cleanup to make safe decisions.
- Workflow test doubles and automated tests — should assert cleanup plan contents and post-cleanup expectations, not only that a command was invoked.

**Acceptance criteria:**

- [ ] A default successful NFS shared-mount Acceptance Test run removes the generated Operational VMs and generated VM inventory artifacts.
- [ ] The same run removes the marked `acceptance-nfs-demo` Ephemeral Dataset.
- [ ] The same run removes the fortress-owned `fortress-nfs-acceptance-nfs-demo-read-write` Derived NFS Share.
- [ ] Cleanup does not delete adopted Datasets, unmanaged Shares, or unrelated fortress-owned Shares.
- [ ] If the Dataset or Share is missing the expected fortress ownership marker, cleanup preserves it and reports a clear blocking reason.
- [ ] If cleanup fails after an earlier workflow failure, the operator sees both the original failure and the cleanup failure.
- [ ] Automated tests cover the cleanup plan/postcondition behavior, including a leftover Dataset, a leftover Share, an unmarked Dataset, and an unrelated fortress-owned Share.

**Out of scope:**

- Changing the Dataset lifecycle model.
- Changing ordinary NAS Reconcile behavior for adopted Datasets.
- Broad live infrastructure CI beyond this NFS shared-mount Acceptance Test cleanup path.
