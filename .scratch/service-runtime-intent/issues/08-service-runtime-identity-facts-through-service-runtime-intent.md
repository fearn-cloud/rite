# Service Runtime Identity facts through Service Runtime Intent

Status: done
Category: enhancement

## What to build

Move **Service Runtime Identity** into **Service Runtime Intent** as resolved fortress-owned runtime names and paths. The fleet-wide analysis should expose the identity facts that Quadlet rendering and Service Deploy currently derive from raw Service yaml: Podman container names, systemd unit names, Service Network names, Container Alias values, Service Data Directory paths, Service Secret names, required mount unit names, and ordered start/stop units.

This issue should add facts and direct tests first. Adapter call sites may continue to use existing fallback traversal until later issues migrate them.
