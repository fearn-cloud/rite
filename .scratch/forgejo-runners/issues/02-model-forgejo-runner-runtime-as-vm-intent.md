# Model Forgejo Runner Runtime as VM-level intent

Status: ready-for-agent
Category: enhancement

## What to build

Teach VM Inventory to express the Forgejo Runner Runtime as VM-level configuration on the Forgejo Runner VM. The intent should capture the Forgejo target, instance-level scope, explicit capability labels, one concurrent job, cleanup policy, and the phase-one rule that job containers do not receive container-runtime socket access by default.

## Acceptance criteria

- [ ] VM Inventory schema accepts a Forgejo Runner Runtime declaration only as VM-level intent.
- [ ] The runtime declaration can name the Forgejo Service or otherwise resolve the Forgejo instance it registers with.
- [ ] The runtime declaration supports instance-level runner scope.
- [ ] The runtime declaration requires explicit runner labels rather than implicit catch-all execution.
- [ ] The runtime declaration supports a one-job concurrency setting and the first Runner VM declares one concurrent job.
- [ ] The runtime declaration captures workspace/cache cleanup policy.
- [ ] The runtime declaration defaults to no container-runtime socket exposure inside job containers.
- [ ] Tests prove valid runner intent is accepted and malformed or ambiguous runner intent is rejected.
