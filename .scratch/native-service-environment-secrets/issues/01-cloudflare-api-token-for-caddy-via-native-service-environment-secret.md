## What to build

Provide Caddy with the Cloudflare API Token through the existing per-Service Sibling SOPS File route, without committing the token in plaintext or routing it through Quadlet Podman secret semantics.

The completed slice should introduce **Native Service Environment Secrets** for Native Services, use the structured `secrets.<purpose>.{created,version,value}` SOPS shape, decrypt values only during Service Deploy, and render the Cloudflare token into Caddy's root-owned environment file on the Ingress Backend VM.

## Agent Brief

**Category:** enhancement

**Summary:** Add Native Service Environment Secret support and use it to feed `CLOUDFLARE_API_TOKEN` to Caddy.

**Current behavior:**

Generated Caddy routes already reference `{$CLOUDFLARE_API_TOKEN}` for Let's Encrypt DNS-01 via Cloudflare, but `internal-ingress` has no declared secret route for that environment variable. Existing **Service Secrets** are Quadlet-oriented: they are installed as Podman secrets and consumed by containers through application-native `_FILE` environment variables. `internal-ingress` is a Native Service, so that path does not fit Caddy.

**Desired behavior:**

Native Services can declare `deploy.environment_secrets`, mapping a structured entry in the Service's Sibling SOPS File to an environment variable name used by a Native config template.

Example declaration:

```yaml
deploy:
  type: native
  config_files:
    - template: caddy.env.j2
      dest: /etc/default/caddy
      mode: "0600"
      restart_on_change: true
  environment_secrets:
    - secret: secrets.cloudflare_api_token
      env: CLOUDFLARE_API_TOKEN
```

Example SOPS shape:

```yaml
secrets:
  cloudflare_api_token:
    created: 2026-05-14T00:00:00Z
    version: 1
    value: "<token bytes>"
```

`service-deploy` should preflight declared Native Service Environment Secrets locally before invoking the remote playbook. Missing entries, scalar legacy entries, and entries missing `created`, `version`, or `value` should fail without printing secret values.

During Native Service Deploy, Ansible should decrypt only the nested `value` entries needed for the selected Service, expose them to templates as a secret map such as `fortress_native_environment_secrets`, and prevent secret values from appearing in logs or diffs. The Caddy environment template should explicitly render:

```jinja
CLOUDFLARE_API_TOKEN={{ fortress_native_environment_secrets.CLOUDFLARE_API_TOKEN }}
```

The actual token must live only encrypted in `inventory/services/internal-ingress.sops.yaml` in the repo, and as plaintext only on the Ingress VM in `/etc/default/caddy` or equivalent runtime state.

**Key interfaces:**

- Native Service inventory schema: accepts `deploy.environment_secrets` only for `deploy.type: native`.
- Inventory validation: rejects `environment_secrets` on Quadlet Services and catches references that are not in the Service's Sibling SOPS namespace.
- Service Deploy preflight: validates structured Native Service Environment Secret entries before remote execution.
- Native Service Deploy playbook: decrypts values during deployment, makes them available to templates, renders secret-bearing files with restrictive mode, and suppresses secret logging.
- `internal-ingress` inventory: declares `secrets.cloudflare_api_token` mapped to `CLOUDFLARE_API_TOKEN`.
- `internal-ingress` Sibling SOPS File: stores `cloudflare_api_token` with `created`, `version`, and encrypted `value`.

## Acceptance criteria

- [ ] Native Service schema accepts `deploy.environment_secrets` entries with `secret` and `env`.
- [ ] Quadlet Service schema or cross-file validation rejects `deploy.environment_secrets`.
- [ ] Validation rejects Native Service Environment Secret references outside the selected Service's Sibling SOPS namespace.
- [ ] `service-deploy` requires a Service Sibling SOPS File when a Native Service declares `environment_secrets`.
- [ ] `service-deploy` preflights Native Service Environment Secrets against the decrypted Sibling SOPS File before invoking the remote Ansible playbook.
- [ ] Preflight errors identify missing or malformed Native Service Environment Secret entries without printing secret values.
- [ ] Scalar legacy entries are rejected for Native Service Environment Secrets.
- [ ] Native Service Deploy exposes decrypted values to templates without passing plaintext through repo-side generated files or checked-in inventory.
- [ ] Secret-bearing Native config rendering uses `no_log` or equivalent protection against Ansible log/diff leaks.
- [ ] `internal-ingress` declares `secrets.cloudflare_api_token` mapped to `CLOUDFLARE_API_TOKEN`.
- [ ] `inventory/services/internal-ingress.sops.yaml` exists and stores `secrets.cloudflare_api_token.created`, `version`, and encrypted `value`.
- [ ] `/etc/default/caddy` rendering is declared with mode `0600` and restarts Caddy on change.
- [ ] Generated Caddy routes continue to reference `{$CLOUDFLARE_API_TOKEN}` and do not contain the token value.
- [ ] Relevant schema, cross-file validation, workflow, and ingress generation tests pass.
