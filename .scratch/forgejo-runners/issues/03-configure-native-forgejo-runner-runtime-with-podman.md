# Configure the Native Forgejo Runner Runtime with Podman

Status: ready-for-agent
Category: enhancement

## What to build

Make VM Configure converge the Forgejo Runner VM into a working Native Forgejo Runner Runtime. The Runner VM should have a dedicated runner Unix user, a VM-local Forgejo runner daemon managed by systemd, Podman available through that runner user's socket, and cleanup behavior that keeps workspaces disposable while preserving no first-class per-repo cache contract.

## Acceptance criteria

- [ ] VM Configure installs the required Forgejo runner runtime packages or binary on a VM that declares Forgejo Runner Runtime intent.
- [ ] VM Configure creates and uses a dedicated runner Unix user.
- [ ] VM Configure enables the runner user's Podman socket and required linger/session behavior.
- [ ] VM Configure renders runner configuration from declared VM-level intent, including labels and one-job concurrency.
- [ ] VM Configure manages the Forgejo runner as a Native systemd service.
- [ ] VM Configure installs scheduled cleanup for disposable workspaces and conservative container cache cleanup.
- [ ] Job containers can be launched through the runner user's Podman socket without exposing that socket into job containers by default.
- [ ] Automated tests cover rendered configuration, systemd unit behavior, Podman socket setup, cleanup configuration, and no Service modeling for job containers.

## Blocked by

- Model Forgejo Runner Runtime as VM-level intent
