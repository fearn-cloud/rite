# Extract the NFS shared-mount Acceptance Test Lifecycle tracer bullet

## What to build

Extract the common Acceptance Test Lifecycle machinery behind the NFS shared-mount Acceptance Test into an importable Python module, then migrate `acceptance-nfs-shared-mount` to use it while preserving the current operator-facing behavior.

The new lifecycle Module should own the shared path for Acceptance Tests that create disposable Operational VMs and Ephemeral Datasets: resolving Host, Template, NAS Endpoint, and Acceptance Policy intent; writing common generated Dataset and VM artifacts; provisioning generated VMs; running NAS Reconcile create/destroy; verifying Derived Share and cleanup postconditions; running retrying SSH checks; preserving resources on `keep_on_fail=true`; and cleaning up generated shared infrastructure by default.

The NFS shared-mount workflow should keep only its workflow-specific proof semantics: asserting that the Primary Acceptance VM and Peer Acceptance VM can read, write, and delete through the shared Mount.
