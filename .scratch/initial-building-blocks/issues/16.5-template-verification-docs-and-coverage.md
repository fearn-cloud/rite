# Template verification docs and coverage

## Parent

.scratch/initial-building-blocks/issues/16-template-verification-workflow.md

## Agent Brief

**Category:** enhancement
**Summary:** Document the complete Template verification operator workflow and add full fake-tool coverage across the combined path.

**Current behavior:**
The Template runbook documents building Templates, but it does not yet document verifying a selected Template on one Host or all Hosts. The split Template verification implementation should leave behind focused tests in each slice, but the combined operator path still needs final coverage that proves the pieces work together without exercising real Proxmox in the fast test suite.

**Desired behavior:**
The operator documentation should explain when to run `templates-build`, when to run `template-verify`, how to verify one Host, how to verify all Hosts, how skipped Hosts are reported, and how `keep_on_fail=true` preserves the failed Template Verification VM for inspection.

The fast automated test suite should use fake tools to cover the complete workflow wiring. Real Proxmox provisioning must remain behind the explicit operator command and outside the default fast test suite.

**Key interfaces:**
- Template runbook — documents Template build and verification as separate operator workflows.
- `template-verify` command — examples for single Host, all Hosts, and failure inspection.
- Fake-tool workflow tests — exercise the combined path without contacting real Proxmox.

**Acceptance criteria:**
- [ ] `runbooks/new-template.md` documents `templates-build` as the separate Template creation workflow.
- [ ] The runbook documents `just template-verify host=<host> template=<template>`.
- [ ] The runbook documents `just template-verify host=all template=<template>`.
- [ ] The runbook explains that `skipped` means a Host does not declare the selected Template.
- [ ] The runbook documents default cleanup on failure.
- [ ] The runbook documents `keep_on_fail=true` and what artifacts are preserved for inspection.
- [ ] Fake-tool tests cover single-Host success across the combined workflow.
- [ ] Fake-tool tests cover `host=all` passed, failed, and skipped summaries.
- [ ] Fake-tool tests cover existing temporary artifact refusal.
- [ ] Fake-tool tests cover Proxmox Template preflight failure.
- [ ] Fake-tool tests cover verification failure with default cleanup.
- [ ] Fake-tool tests cover verification failure with `keep_on_fail=true`.
- [ ] Fake-tool tests cover cleanup failure reporting.
- [ ] The real Proxmox provisioning path remains outside the default fast test suite.

**Out of scope:**
- Implementing missing policy, generation, orchestration, or verification behavior from earlier child issues.
- Running a real Template verification demo on a Host.
- Auto-building Templates as part of verification.
