## What to build

The transition from a freshly-installed Proxmox host carrying a shared bootstrap key to a host with a unique per-host SSH key stored encrypted in the repo. Idempotency is by refusal — re-running on a bootstrapped host fails rather than clobbers.

## Acceptance criteria

- [x] Bootstrap playbook generates a per-host SSH keypair locally
- [x] Pushes public key to host via shared bootstrap key
- [x] Verifies new key works (auth-test) before proceeding
- [x] Removes shared key from host's `authorized_keys`
- [x] Writes encrypted private key into `inventory/hosts/<host>.sops.yaml` as a structured key entry (type, created, public_key, private_key)
- [x] Refuses to run if `<host>.sops.yaml` already contains a bootstrap key entry
- [x] `just host-bootstrap host=<name>` exposes the workflow
- [x] `runbooks/new-host.md` documents the bootstrap step
- [x] Demo: fresh PVE install of wintermute transitioned end-to-end
